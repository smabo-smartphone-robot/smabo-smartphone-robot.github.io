コンポーネント詳細

📱

smabo-app

Flutter / Dart

ESP32 ロボット用スマートフォンアプリ。スマホがロボットの「顔」になりつつ、 走行コントローラ・アームコントローラ・センサパブリッシャ・音声インタフェースを兼ねます。

機能概要

機能説明
顔表示カスタマイズ可能な2Dアニメーション顔（表情・色・形・まぶた・回転）
走行コントロール仮想ジョイスティックで /cmd_vel 送信
アームコントロールサーボ角度指令 /servo/command 送信
センサ送信IMU・GPS・カメラ映像を brain デバイスへ送信
音声インタフェースウェイクワード「smabo」で STT 起動、TTS でロボットが発話
表情切替/expression トピック（Int32）でリモート切替可能

アーキテクチャ

スマートフォン (smabo-app)
│
├── UI層
│   ├── 顔ページ      — 表情アニメーション、カスタマイズ設定
│   ├── 走行ページ    — 仮想ジョイスティック
│   └── アームページ  — サーボ操作UI
│
├── State層
│   └── AppState (Provider)
│
└── Core層
    ├── wire/ws_client.dart     — ROS非依存WebSocketクライアント
    ├── wire/ros_compat.dart    — rosbridgeフォーマット互換レイヤ（ここだけ）
    └── models/                 — FaceConfig, FaceExpression など

接続先（設定画面で変更可）

エンドポイントデフォルト用途
ESP32ws://<ip>:9090走行・サーボ・設定同期
brain デバイスws://<ip>:9090顔追従・音声・カメラ
🔗 ESP32 へ: /cmd_vel, /servo/command, set_mode, set_config, get_config（受信は /odom, /joint_states）
brain へ: /speech/recognized, /imu/data, /gps/fix, カメラ映像（受信は /look_at, /speech/say, /expression）

開発環境

Flutter 3.44.1 Dart SDK >=3.2.0 <4.0.0 Android iOS

主要な依存パッケージ

provider — 状態管理 web_socket_channel — 通信 shared_preferences — 設定永続化 speech_to_text — STT flutter_tts — TTS camera sensors_plus — IMU geolocator — GPS permission_handler vector_math flutter_colorpicker image_picker

smabo-esp32

MicroPython

ESP32 用ロボットファームウェア。スマホアプリからの WebSocket JSON 命令を受け取り、 走行・サーボ・オドメトリを制御します。

動作環境

ESP32（DevKit / S3 / XIAO ESP32S3） MicroPython v1.22 以降 PCA9685（サーボ） TB6612FNG（DCモータ）

セットアップ

① MicroPython を書き込み、② WiFi 設定ファイルを作成、③ ファイルを転送、の流れです。

# ① MicroPython を書き込む
esptool.py --port /dev/ttyUSB0 erase_flash
esptool.py --port /dev/ttyUSB0 write_flash -z 0x1000 ESP32_GENERIC-*.bin

# ② ボードに合わせて設定テンプレートをコピー（SSID/password を編集）
cp configs/config.esp32-classic.json config.json

# ③ ファイルを転送（mpremote の例）
mpremote connect /dev/ttyUSB0 cp config.json :
mpremote connect /dev/ttyUSB0 cp *.py :
🔑 config.json には WiFi の SSID / パスワードを記載します。リポジトリにはコミットしないよう注意してください（省略した設定は config.py の DEFAULTS が使われます）。

モジュール構成

ファイル役割
main.py起動エントリ・asyncio イベントループ
config.py永続設定（RAM + config.json、デバウンス保存）
wifi_manager.pyWiFi 接続・自動再接続
ws_server.pyRFC 6455 WebSocket サーバ（外部ライブラリ不要）
robot.pyオーケストレータ（rosbridge プロトコル・モード管理）
pca9685.pyPCA9685 PWM ドライバ（サーボ用 I2C）
servo_controller.pyJointGroup（全サーボ共通）
random_motion.pyグループ単位のランダム動作
dc_motors.pyTB6612 差動駆動（cmd_vel 受信・デッドマン停止）
encoder.pyGPIO 割り込みによるエンコーダカウント
odometry.py差動駆動オドメトリ → nav_msgs/Odometry

設定テンプレート

ファイル対象ボード
config.esp32-classic.jsonESP32 DevKit（38ピン標準）
config.esp32s3-devkitc1.jsonESP32-S3 DevKitC-1
config.xiao-esp32s3.jsonSeeed XIAO ESP32S3

通信プロトコル

WebSocket ws://<ESP32-IP>:9090 で JSON 送受信。フォーマットは rosbridge v2.0 互換。認証なし（信頼済み LAN 内利用前提）。

方向トピック内容
受信/cmd_vel走行速度指令
受信/servo/commandサーボ軌道指令
送信/odomオドメトリ
送信/joint_states関節角度

smabo-brain

任意 / オプション

PC（ラズパイなどの SBC でも可）に載せるソフト。画像処理など、スマホや ESP32 では処理が重いタスクを担当し、 高度な自律制御を実現します。必須ではなく、必要に応じて追加できる中間レイヤです。

🚧 smabo-brain は開発中のコンポーネントです。最新の状況やドキュメントはリポジトリをご確認ください。
